    什么？？？都2202年了，你还不懂原型和原型链？？？

    growydp 2/22/2022

    # 哈哈哈，标题开个玩笑

    现在大多数老司机都听过或者用过JavaScript的原型技术（比如继承），但是在面试的时候还是很难用语言表达出来！

    我 ... 我有一个朋友就是这种情况 ...那么本篇文章，将帮助还没有了解过或者想巩固原型知识的开发者们（包括我那位朋友）。

    # 原型

    JavaScript被描述为一种基于原型的语言，每个对象（null除外）都拥有一个原型对象。以其原型为模板，从原型继承方法和属性。-MDN

    # 原型链

    原型对象也可能拥有原型，并从中继承方法和属性，一层一层，以此类推。这种关系常被称为原型链（prototype chain）。-MDN

    上面这条概念帮助我们理解了为什么我们可以使用数组的concat/filter/push等方法，因为这些方法都位于Array构造函数的原型上。

    Array.prototype.concat()
Array.prototype.filter()
Array.prototype.push()

    # 原型链查找规则

    在一个对象中查找给定属性的值，首先从当前对象中查找。如果找到给定属性则返回属性的值，如果没有找到则查找对象的原型。如果对象的原型中也没有找到则顺着对象的原型链继续向上查找，直到找到原型链顶端。如果还没有找到则返回undefined。

    在查找规则中提到了原型链顶端，稍后将详细介绍。看不懂没关系，泡一杯茶，多看几遍。

    # 必备知识，否则学完也是一团糟

    现在，我们将了解几个概念，它们将帮助我 ... 我朋友和在座的各位前端大神更好的理解原型。

    # prototype

    上面的图可以用以下代码解释

    function Person() {
  // TODO
}

Person.prototype.name = 'growydp'

    用通俗的语言解释：构造函数的prototype属性指向了一个对象，该对象是调用构造函数而创建的实例的原型。

    # __proto__

    每个实例对象（object）都有一个私有属性（称之为__proto__）指向它的构造函数的原型对象（prototype）。-MDN

    懂了！！！我是老懂王！！！话不多说，接着奏乐接着上图。

    但是，但是，但是！！！切记！！！Object.prototype.__proto__已经从Web标准中删除，虽然一些浏览器目前支持它，但是在将来某个时间停止支持，请尽量不要使用该属性。接下来上代码！

    function Person() {
  // TODO
}

Person.prototype.name = 'growydp'

const p = new Person()

// 上面已经解释过了
// 线上环境不要这样写
console.log(p.__proto__ === Person.prototype) // true

    遵循ECMAScript标准，someObject.[[Prototype]]符号是用于指向someObject的原型。从ES6开始，[[Prototype]]可以通过Object.getPrototypeOf和Object.setPrototypeOf访问器来访问

    console.log(Object.getPrototypeOf(p) === Person.prototype) // true

    接下来，我们填上面留下的一个坑。原型链的顶端是什么？

    console.log(Object.getPrototypeOf(Object.prototype)) // null

    根据定义，null没有原型，并作为这个原型链中的最后一个环节

    # constructor

    上面介绍了：

    1. 构造函数与由该构造函数创建的实例的原型之间的关系
    2. 构造函数创建的实例与构造函数原型之间的关系

    下面将介绍由该构造函数创建的实例的原型与构造函数的关系（说实话，我朋友被绕进去了 手动狗头）

    function Person() {
  // TODO
}

Person.prototype.name = 'growydp'

const p = new Person()

console.log(Person.prototype.constructor === Person) // true

    用通俗的语言解释：由构造函数创建的对象的原型通过constructor指向构造函数。这个特性可以用于判断某个变量的类型，比如

    console.log([].constructor === Array) // true

    诶，不对啊，很明显[]是构造函数Array的实例啊，不应该是Array.prototype的constructor指向Array吗？诸君，请往下看！

    # 实例与构造函数的关系

    直接上答案，其实上面的代码完整版是这样的：

    console.log([].__proto__.constructor === Array) // true

    因为前面我们一直在说，原型，就是实例继承原型上的属性和方法。步骤如下

    1. 首先从实例[]中查找constructor属性，没有找到，查找其原型
    2. 在原型上Person.prototype找到了constructor属性，该属性指向Person构造函数，查找结束，返回结果。
