    作用域

    growydp 4/23/2022

    # 什么是作用域？

    作用域就是负责收集并维护由所有声明的标识符（变量）组成的一系列查询，实施一套非常严格的规则，确定当前执行的代码对这些标识符的访问权限。-《你不知道的JavaScript》

    如果一个变量或者其他表达式不在当前的作用域中，那么它就是不可用的。作用域可以根据代码层次分层，以便子作用域可以访问父作用域，通常是指沿着链式的作用域链查找，不能从父作用域引用子作用域中的变量和引用。。-MDN

    上面我们提到了作用域可以根据代码层次分层，其实就是当一个块或函数嵌套在另一个块或者函数中时，就发生了作用域的嵌套。

    # 静态和动态作用域

    静态（词法）作用域：函数的作用域在函数定义时就决定了。

    JavaScript采用的就是静态（词法）作用域，也就是函数的作用域在函数定义时就决定了。

    var name = 'growydp'
function sayName() {
  console.log(name)
}
sayName() // 'growydp'

    上面代码会首先在sayName函数内查找变量name，在函数内没有找到，就会从词法层面（函数定义时）的父级查找。上面函数sayName的父级是全局作用域，找到了变量name并返回结果。

    下面我们修改一下代码：

    var name = 'growydp'
function sayName() {
  console.log(name)
}
function person() {
  var name = 'jone'
  sayName()
}
person() // 'growydp'

    根据我们前面的解释，就会很快得到上面代码的答案。

    无论函数在哪里被调用，也无论它如何被调用，它的词法作用域都只由函数被声明时所处的位置决定。

    动态作用域：函数的作用域是在函数调用的时候才确定的。

    # 作用域链查找规则

    上面我们简单介绍了作用域链查找规则，其实就是在当前作用域中查找给定变量，如果找到给定变量直接返回，否则就从这个作用域词法层面的父级作用域中继续查找该变量。以此类推，直到找到该变量，或者到达最外层的作用域（全局作用域为止）。

    遮蔽效应

    作用域查找会在找到第一个匹配的标识符时停止，在多层嵌套作用域中可以定义同名的标识符（内部标识符遮蔽了外部的标识符）。

    # 全局作用域

    在代码初始化的时候，全局变量就存在于全局作用域中。不管在全局作用域、函数作用域还是块作用域中都可以访问全局作用域的变量。

    # 全局变量和全局函数

    定义在全局作用域中的变量和函数就会成为全局变量和全局函数

    # 意外成为全局变量

    LHS和RHS

    当引擎执行LHS查询时，如果在全局作用域中也无法找到给定变量，在非严格模式下，全局作用域中就会创建一个具有该名称的变量。（严格模式下禁止自动或隐式地创建全局变量）

    function sayAge() {
  age = 23
}
sayAge()
console.log(age) // growydp
console.log(window.age) // growydp

    上面在函数作用域中定义的变量意外的变成了全局变量，接下来我们测试一下在严格模式下的情况。

    + 'use strict';
function sayAge() {
  age = 23
}
sayAge()
console.log(age) // Uncaught ReferenceError: age is not defined
