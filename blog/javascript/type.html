    # 基本数据类型

    # Number类型

    # IEEE 754

    数值类型，实现借用自IEEE 754标准，该标准来表示整数和浮点数值。IEEE 754全称叫做IEEE 二进制浮点数算术标准，这个标准定义了表示浮点数的格式等内容。

    浮点数思想

    把一个数拆成两部分来存储：

    1. 有效位数，有时候也叫系数、分数或者尾数
    2. 指数，表示小数点应该插在系数的哪个位置。

    浮点数的实现非常复杂，需要在有限的存储位数中用一种固定的存储格式在数字精度和数字范围之间做权衡。

    JavaScript的浮点数并不是IEEE 754标准的完全实现。Java的浮点数实现用的是IEEE 754的一个子集，JavaScript用的是该子集的子集，JavaScript的number类型与Java的 double类型非常相似，都是64位的浮点数类型。一个number类型包含：

    • 1位符号位，如果该位为1则说明该数是负数
    • 11位指数位
    • 53位有效位数

    # 0

    在IEEE 754标准中有两个零：+0和-0

    console.log((1/0) === (1/-0)) // false
console.log(Object.is(0, -0)) // false

    # 范围

    如果某个数值超出JavaScript可以表示的范围，这个数值会被自动转换为一个无穷值（+Infinity或者-Infinity）。Infinity指那些大到无法表示的值。具体范围：

    • Number.MIN_VALUE：ECMAScript可以表示最小数值
    • Number.MAX_VALUE：ECMAScript可以表示最大数值

    可以使用isFinite()或者Number.isFinite()判断一个值是不是有限大。前者会将测试的值强转，后者不会。他们会在值为NaN、Infinity或者-Infinity的时候返回false

    • Number.NEGATIVE_INFINITY表示负无穷大，其特性MDN 负无穷大 (opens new window)
    • Number.POSITIVE_INFINITY表示正无穷大，其特性MDN 正无穷大 (opens new window)

    可以使用Number.isSafeInteger()判断一个值是否是安全整数。

    • Number.MAX_SAFE_INTEGER，表示JavaScript中最大安全整数MDN 最大安全整数 (opens new window)
    • Number.MIN_SAFE_INTEGER，表示JavaScript中最小安全整数MDN 最小安全整数 (opens new window)

    Number.isInteger()

    使用Number.isInteger()函数判断一个数是否是整数，不管它是不是安全整数。

    所有比Number.MAX_SAFE_INTEGER大的数还是被称为整数。虽然有一些可能是整数，但大部分不是

    # NaN（Not a Number）

    NaN是一个特殊值，用于表示那些非数的数，使用typeof判断该值返回number！！！！

    • 当字符串转换成数值失败时，结果就是NaN。
    • 当算术表达式中的一个数值为NaN的时候，它的运算结果也会是NaN。（任何涉及NaN的操作始终返回NaN）
    • NaN不等于包括NaN在内的任何值。

    判断一个值是NaN的方法：

    • isNaN()：如果当前值是NaN，或者将其强制转换为数字后将是NaN，返回true
    • Number.isNaN()：仅当值当前为NaN时才为true

    isNaN() 判断对象

    在测试对象的时候，首先会调用对象的valueOf()方法，然后确定返回的值是否可以转换为数值。如果不能，调用toString()方法，并测试其返回值。

    # Number

    可以返回number类型值的函数。number类型在JavaScript中是不可变类型。不应该在Number函数前加上new

    const n = Number("7")
const num = new Number("7")

console.log(typeof n) // number
console.log(typeof num) // object
console.log(n === num) // false

    Number.EPSILON

    是JavaScript中最小的正数，将其加1，会得到一个大于1的值。任何比Number.EPSILON小的正数加1的结果还是1。

    Number.EPSILON (opens new window)

    当计算一堆数的和时，相加的顺序会影响结果。(0.1+0.2)+0.3的结果比0.1+(0.2+0.3)的结果大。

    # 运算符

    前缀运算符

    运算符 作用 描述
    + 转为数值类型 自动将数据类型进行了转换，转为数值类型，如果转换失败，结果就是NaN
    - 反转正负 改变目标值的正负
    typeof 返回类型 如果目标是一个数值类型，返回的结果是“number”

    中缀运算符

    运算符 作用 描述
    + 加法运算 JavaScript中，+不仅可以做数值的加法运算，还可以做字符串的拼接。两值相加的时候，如果一个值是字符串，那么另一个值也会被转换为字符串进行拼接
    - 减法运算
    * 乘法运算
    / 除法运算
    % 取余运算 JavaScript不存在取模运算，只有取余运算。前者正负号取决于被除数，后者正负号取决于除数
    ** 幂运算
