    我朋友说他不会浏览器的本地存储

    vuePress-theme-reco    2022

    我朋友说他不会浏览器的本地存储

    growydp 2/22/2022

    # 简介

    在我日常开发过程中，经常使用cookie保存用户token。至于其他缓存我还真的没怎么用过，还好在写这篇文章之前没人问过我浏览器缓存有哪些？说一说他们的优点和缺点？不然作为高级 ... 初级前端工程师的我连这么基础的概念都不了解。，岂不是很 ...

    浏览器的本地存储，是由JavaScript APIs组成的。可以存储可读取需要的数据。这样做有很多好处：

    • 个性化网站偏好
    • 保存之前的站点行为
    • 本地换保存数据和静态资源可以使一个站点更快（让资源变少）的下载，甚至可以在网络失去链接的时候变得暂时可用。
    • 保存web已经生产的文档可以在离线状态下访问。

    使用客户端存储API可以存储的数据量是有限的，具体的数量限制取决于浏览器。

    # Cookie

    Cookie最开始设计出来并不是来做本地存储的，为了弥补HTTP在状态管理上的不足。但是Cookie已经过时了，存在各种安全问题，无法存储复杂数据。它的优势是它们得到了非常旧的浏览器的支持，所以如果项目需要支持IE8或者更早的浏览器，Cookie绝对是首选。

    Cookie本质是浏览器里面存储的一个很小的文本文件，内部以键值对的方式来存储。向同一个域名下发送请求，都会携带相同的Cookie，服务器拿到Cookie进行解析，便能拿到客户端的状态。Cookie的缺点：

    • 容量缺陷：体积上限只有4kb，只能存储有限的信息
    • 性能缺陷：Cookie紧跟域名，不管域名下面的某一个地址需不需要这个Cookie，请求都会携带上完整的Cookie。随着请求数量增多，会造成巨大的性能浪费，因为携带了很多不需要的内容。
    • 安全缺陷：以纯文本形式在浏览器和服务器中传递，很容易被非法用户截获。

    # localStorage

    存储的数据将被存在浏览器会话中，在同一个域名下，会存储相同的一段localStorage

    • 容量：localStorage的容量上限为5M
    • 性能和安全：只存在客户端，默认不参与与服务端的通信。
    • 接口封装：通过localStorage暴露在全局
      • setItem:设置localStorage.setItem('name', 'growydp')
      • getItem:获取const name = localStorage.getItem('name')
      • removeItem:删除某一个localStorage.removeItem('name')
      • clear:清空localStoragelocalStorage.clear()

    在日常开发中，经常将上述方法封装起来，方便对即将存入localStorage的数据进行格式化处理。比如判断传入数据如果是对象，则调用JSON方法将其转成字符串保存

    function handleSet(key, value) {
  if (typeof value === 'object' && value !== null) {
    value = JSON.stringify(value)
  }
  localStorage.setItem(key, value)
}

function handleGet(key) {
  const value = localStorage.getItem(key)
  try {
    return JSON.parse(value)
  } catch (_) {
    return value
  }
}

    # sessionStorage

    sessionStorage属性允许访问一个当前源的session Storage对象。与localStorage相似，不同之处在于localStorage里面存储的数据没有过期时间设置，存储在sessionStorage里面的数据在页面会话结束时会被清除。

    • 容量，sessionStorage的容量上限也是5M
    • 性能与安全：只存在客户端，默认不参与与服务端的通信
    • 接口封装：与localStorage方法一样
      • setItem:设置localStorage.setItem('name', 'growydp')
      • getItem:获取const name = localStorage.getItem('name')
      • removeItem:删除某一个localStorage.removeItem('name')
      • clear:清空localStoragelocalStorage.clear()

    特点

    • 页面会话在浏览器打开期间一直保持，重新加载或恢复页面仍会保持原来的页面会话。
    • 在新标签或窗口打开一个页面时会复制顶级浏览会话的上下文作为新会话的上下文，与session、cookies的运行方式不同。
    • 打开多个相同的URL的Tabs页面，会创建各自的sessionStorage
    • 关闭对应浏览器标签或窗口，会清除对应的sessionStorage

    存储在sessionStorage和localStorage中的数据特定于页面的协议

    http和https的sessionStorage相互隔离。对于对象、整数key值会自动转换成字符串形式。
