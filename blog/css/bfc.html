    令人头疼的BFC究竟是什么？

    growydp 2/23/2022

    # BFC同学与面试官对话

    • 面试官：请在3分钟内简单介绍一下自己。
    • BFC：这个我熟，等我找找MDN上关于我的介绍...
    • 面试官：...

    BFC（块格式化上下文 Block Formatting Context）是Web页面的可视CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。

    • 面试官：额...好吧，请用大白话介绍一下你自己...

    # BFC规则

    1. BFC元素的子元素会在垂直方向上一个接一个的放置
    2. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响容器外的元素
    3. 垂直方向上的距离由margin决定，属于同一个BFC的两个相邻元素的外边距会重叠（与方向无关）
    4. 每个元素的左外边距与包含块的左边界相接触（对于从右到左的格式化，右外边距接触包含块的右边界），浮动元素也是一样，说明BFC中的子元素不会超出它的包含块，除非这个子元素也创建一个新的BFC（自身也是一个浮动元素）
    5. 计算BFC的高度时，浮动子元素也参与计算
    6. BFC的区域不会与float的元素区域重叠
    • 面试官：...我就说我怎么好像见过你小子，原来margin重叠就是你小子搞的鬼
    • BFC：正是在下！
    • 面试官：那你再说说什么情况下还能看见你？

    # BFC触发条件

    • 根元素：HTML标签就可以触发BFC，里面的元素就遵循BFC的规则
    • 浮动元素：float值不是none（常见就是left和right）
    • 绝对定位元素：position为absolute或者fixed
    • display：值为inline-block/table/flex/grid/inline-flex/inline-grid/table-cell/table-caption/table-row/table-row-group/table-header-group/table-footer-group/inline-table
    • overflow：不为visible

    # BFC同学的技能

    • 面试官：说了那么多，你究竟能够解决什么疑难杂症呢？说说你的优点
    • BFC：优点？那可太多了！

    # margin重叠

    • BFC：就拿你刚才说见过我那次举例
    .box1 {
  background: pink;
  width: 100px;
  height: 100px;
  margin: 15px;
}

.box2 {
  background: purple;
  width: 100px;
  height: 100px;
  margin: 15px;
}

    <div class="box1"></div>
<div class="box2"></div>

    • BFC：原因我在上面也给你介绍过了，那么怎么解决呢？属于同一个BFC的元素外边距会重叠，那么解决方法就是让它们不属于同一个BFC就可以了....

    方法一

    在第二个div，也就是box2套一个外壳

    <div class="box1"></div>
<div style="overflow: hidden">
  <div class="box2"></div>
</div>

    方法二

    激活第二个box为一个新的bfc

    .box2 {
  background: purple;
  width: 100px;
  height: 100px;
  margin: 15px;
  float: left;
}

    # 浮动元素与BFC盒子不重叠

    .box1 {
  background: pink;
  width: 100px;
  height: 100px;
  float: left;
}

.box2 {
  background: purple;
  width: 100px;
  height: 100px;
}

    <div class="box1"></div>
<div class="box2"></div>

    • 面试官：你把我紫色的盒子放哪了？？？
    • BFC：大哥，你没学过脱离文档流吗？
    • 面试官：...其实 那个...
    • BFC：利用上面介绍的规则float元素与BFC元素不重叠的规则，可以解决这个问题
    .box2 {
  background: purple;
  width: 100px;
  height: 100px;
  overflow: hidden;
}

    # 清除浮动

    • 面试官：清除浮动让很多前端初学者很头疼，你有办法解决吗？
    • BFC：of course，根据我的规则，计算BFC高度时，浮动元素也参与计算
    .container {
  width: 200px;
}

.box1 {
  background: pink;
  width: 100px;
  height: 100px;
  float: left;
}

.box2 {
  background: purple;
  width: 100px;
  height: 100px;
  float: left;
}

    • BFC：这时候，我们只需要将外层的元素变为BFC即可
    .container {
  width: 200px;
  overflow: hidden;
}
