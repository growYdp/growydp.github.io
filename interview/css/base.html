    # display有哪些属性？

    名称 描述
    none 此元素不会被显示
    block 显示为块级元素
    inline 显示为内联元素（行内）
    inline-block 显示为行内块元素
    flex flex布局
    grid grid布局
    table 作为块级表格来显示
    inherit 从父元素继承display属性的值

    # display的block、inline和inline-block的区别？

    • block：会独占一行，多个元素会另起一行 可以设置width height margin 和 padding 属性
    • inline：元素不会独占一行，设置width height属性无效，但可以设置水平方向的margin 和 padding属性，不能设置垂直方向的padding 和 margin
    • inline-block：将对象设置为inline对象，但对象的内容作为block对象呈现，之后的内联对象会被排列在同一行内

    # 行内元素和块级元素的区别

    行内元素

    • 设置宽高无效
    • 可以设置水平方向的margin和padding，不可以设置垂直方向的margin和padding
    • 不会自动换行

    块级元素

    • 可以设置宽高
    • 可以设置padding和margin
    • 可以自动换行
    • 多个块状，默认排列顺序从上到下

    # display: none、visibility: hidden、opacity: 0的区别？

    区别 display: none visibility: hidden opacity: 0
    是否占据空间 不占据 占据 占据
    子元素是否继承 不会被子元素继承 会被子元素继承，设置子元素visibility: visible来显示子元素 会被子元素继承，但是不能设置子元素opacity: 1重新显示
    事件绑定 元素都不存在，无法触发绑定的事件 不会触发上面绑定的事件 元素上面绑定的事件可以触发
    过渡动画 无效 无效 有效

    # 扩展 隐藏元素的方法有哪些

    • display: none 渲染树不会包含该渲染对象 因此该元素不会再页面中占据位置 不会响应绑定的事件 会造成文档重排
    • visibility: hidden 元素占据空间，不会响应绑定的监听事件 造成本元素的重绘
    • opacity: 0 元素占据空间，会响应绑定的监听事件
    • position: absolute 使用绝对定位将元素移除可视区域内 实现元素的隐藏
    • z-index: 负值 使其他元素遮盖住该元素，实现隐藏
    • clip/clip-path 元素裁剪的方法实现元素的隐藏 元素在页面中占据位置 但是不会响应绑定的监听事件
    • transform: scale(0, 0) 将元素缩放为0 实现元素的隐藏 元素仍在页面中占据位置，不会响应绑定的监听事件

    # 介绍一下标准的CSS盒子模型？与怪异盒子模型的区别？它们之间怎么切换？

    CSS盒模型分为W3C（标准）盒模型和IE（怪异）盒模型

    • IE盒模型：width和height计算公式为content width + padding + border
    • W3C盒模型：width和height仅仅包含content

    IE盒模型和W3C盒模型切换的方法：

    • box-sizing: content-box切换为W3C盒模型
    • box-sizing: border-box切换为IE盒模型

    # CSS选择器有哪些？哪些属性可以继承？CSS优先级算法？

    CSS选择器：

    • id选择器（0100）：#name/#age/#box
    • 类选择器（0010）：.name/.age/.box
    • 伪类选择器（0010）：a:hover/p:first-child
    • 属性选择器（0010）：a[title]
    • 标签选择器（0001）：div/span/h1
    • 伪元素选择器（0001）：p::first-line
    • 通配符选择器（0000）：*
    • 子代选择器（0000）：div > span/div > h1
    • 后代选择器（0000）：div span/div h1
    • 相邻选择器（0000）：div + h1

    可继承属性

    只有颜色、文字、字体间距、行高、对齐方式和列表的样式可以继承。 例如font-size/font-family/color/line-height等等

    不可继承的样式

    border/padding/margin/width/height

    更多可继承和不可继承 (opens new window)

    优先级

    浏览器通过优先级来判断哪些属性值与一个元素相关，从而在元素上应用这些属性值。优先级是基于不同种类选择器组成的匹配规则。

    权重计算规则：

    1. 内联样式：1000
    2. ID选择器：0100
    3. 类、伪类、属性选择器：0010
    4. 标签选择器、伪元素选择器：0001
    5. 通配符、子选择器、相邻选择器：0000
    6. 继承的样式没有权重

    注意：

    1. !important声明的样式优先级最高，如果冲突再进行计算。（比内联样式优先级还高）
      1. 一定要优先考虑使用样式规则的优先级来解决问题而不是!important
      2. 只有在需要覆盖全站分或者外部CSS的特定页面中使用!important
      3. 永远不要在插件中使用!important
      4. 永远不要在全站范围内的CSS代码中使用!important
    2. 当优先级与多个CSS声明中任意一个声明的优先级相等的时候，CSS中最后的那个声明将会被应用到元素上。
    3. 样式表的来源不同时，优先级顺序为：内联样式>内部样式>外部样式>浏览器用户自定义样式

    # 伪元素和伪类的区别和作用？

    # 伪元素

    在内容元素的前后插入额外的元素或样式，但是这些元素实际上并不在文档中生成。他们只在外部显示可见，但不会在文档的源代码中找到它们。因此称为伪元素

    p::before { content: 'brfore' }
p::after { content: 'after' }
p::first-line { background: red }
p::first-letter { font-size: 30px }

    # 伪类

    将特殊的效果添加到特定选择器上，它是已有元素上添加类别的，不会产生新的元素。

    a:hover { color: red }
p:first-child { color: pink }

    伪类是通过在元素选择器上加入伪类改变元素状态，而伪元素通过对元素的操作进行对元素的改变。

    # position有哪些值？分别简单介绍一下？

    值 描述
    relative 相对定位，不脱离文档流，参考自身静态位置通过top/right/bottom/left定位
    absolute 绝对定位，参考最近一个不为static的父级元素通过top/right/bottom/left定位。可以设置外边距，不会与其他边距合并
    fixed 固定定位，参照对象是可视窗口
    sticky 粘性定位
    static 默认值，没有定位，元素出现在正常的流中
    inherit 从父元素中继承position属性的值

    # link和@import的区别

    都是外部引用css的方式，区别：

    • link是XHTML标签，除了加载CSS外，还可以定义RSS等其他事务 @import属于CSS范畴，只能加载CSS
    • link引用CSS时，在页面载入时同时加载；@import需要页面网页完全载入以后加载
    • link是XHTML标签，无兼容问题；@import是在CSS2.1提出的，低版本的浏览器不支持
    • link支持使用JavaScript控制DOM去改变样式；而@import不支持

    # transition和animation的区别

    • transition是过渡属性，强调过渡，它的实现需要触发一个事件才执行动画（鼠标移动上去，焦点，点击等）。类似于flash的补间动画，设置一个开始关键帧，一个结束关键帧。
    • animation是动画属性，它的实现不需要触发事件 设定好时间后可以自己执行，且可以循环一个动画。也类似于flash的补间动画 它可以设置多个关键帧（@keyframe定义）完成动画。

    # 对requestAnimationFrame的理解

    实现动画效果的方法比较多

    • JavaScript中可以通过定时器setTimeout来实现
    • CSS3中可以使用transition和animation来实现
    • HTML5中的canvas也可以实现
    • HTML5提供一个专门用于请求动画的API：requestAnimationFrame 请求动画帧

    mdn window.requestAnimationFrame (opens new window)

    window.requestAnimationFrame()希望执行一个动画，并且要求浏览器在下次重绘之前调用指定的回调函数更新动画。该方法需要传入一个回调函数作为参数，该回调函数会在浏览器下一次重绘之前执行。

    语法：window.requestAnimationFrame(callback)。callback是下一次浏览器重绘前需要调用的回调函数。该回调函数会被传入DOMHighResTimeStamp参数，它表示requestAnimationFrame开始去执行回调函数的时刻。

    方法属于宏任务，会在执行完微任务之后再去执行。

    **取消动画：**使用cancelAnimationFrame()来取消执行动画，方法接收一个参数requestAnimationFrame默认返回的id，只需要传入这个id就可以取消动画了。

    优势：

    • CPU性能：
      • 使用setInterval实现的动画，当页面被隐藏或最小化时，仍在后台执行动画任务，此时页面处于不可见或不可用状态，刷新动画是没有意义的，完全是浪费CPU资源。
      • requestAnimationFrame不同，当页面处理未激活的状态下，该页面的屏幕刷新任务也会被系统暂停

    # 重排（reflow）和重绘（repaint）的理解

    # 重排

    无论通过什么方式影响了元素的几何信息（元素在视口内的位置和尺寸大小），浏览器需要重新计算元素在视口内的几何属性，这个过程叫做重排。

    # 重绘

    通过构造渲染树和重排（回流）阶段，我们知道哪些节点是可见的，以及可见节点的样式和具体的几何信息（元素在视口内的位置和尺寸大小），接下来就可以将渲染树的每个节点都转换为屏幕上的实际像素，这个阶段叫做重绘。

    # 实现两栏布局（左侧固定 + 右侧自适应布局）

    <div class="outer">
  <div class="left">左侧</div>
  <div class="right">右侧</div>
</div>

    # 第一种（右边元素使用margin-left）

    利用浮动，左边元素宽度固定，设置向左浮动。将右边元素的margin-left设为固定宽度。注意：右边元素的width默认为auto，所以会自动撑满父元素

    .outer {
  height: 100px;
}

.left {
  width: 200px;
  float: left;
  background: pink;
  height: 100%;
}

.right {
  margin-left: 200px;
  background: purple;
  height: 100%;
}

    # 第二种（右边元素使用BFC特性）

    利用浮动，左边元素宽度固定，设置向左浮动。右侧元素设置overflow: hidden，这样右边就触发了BFC，BFC的区域不会与浮动元素发生重叠，所以两侧就不会发生重叠。

    .outer {
  height: 100px;
}

.left {
  width: 200px;
  background: pink;
  float: left;
  height: 100%;
}

.right {
  height: 100%;
  background: purple;
  overflow: auto;
}

    # 第三种（flex）

    .outer {
  height: 100px;
  display: flex;
}

.left {
  width: 200px;
  height: 100%;
  background: pink;
  flex-shrink: 0; // 重要
}

.right {
  height: 100%;
  background: purple;
  flex: 1;
}

    # 第四种（左边元素利用position的绝对定位 右边元素使用margin-left）

    利用绝对定位，父级元素设为相对定位，左边元素absolute定位，宽度固定。右边元素的margin-left的值设为左边元素的宽度值

    .outer {
  position: relative;
  height: 100px;
}

.left {
  position: absolute;
  width: 200px;
  height: 100%;
  background: pink;
}

.right {
  height: 100%;
  background: purple;
  margin-left: 200px;
}

    # 第五种（右边元素绝对定位 left为左边元素的宽度）

    利用绝对定位，父级元素设为相对定位。左边元素宽度固定，右边元素absolute定位，left为宽度大小，其余方向定位为0

    .outer {

  position: relative

  background: pink;
}

.right {

  background: purple;
  position: absolute;
  left: 200px;
  top: 0;
  right: 0;
  bottom: 0;
}

    # 实现圣杯布局和双飞翼布局（经典三分栏布局）

    目的：

    • 三栏布局：中间一栏最先加载和渲染
    • 两侧内容固定，中间内容随着宽度自适应
    • 一般应用于pc网页

    技术总结：

    • 使用float布局
    • 两侧使用margin负值，以便和中间内容横向重叠
    • 防止中间内容被两侧覆盖，圣杯布局用padding，双飞翼布局用margin

  <p class="left">我是左边</p>

</div>

}

  background: pink;

.right {
  width: 150px;
  margin-right: -150px;
  background: green;

    # flex: 1 包含哪些意思？

    • flex-grow: 1：属性默认是0，如果存在剩余空间，元素也不放大，设置为1代表会放大
    • flex-shrink: 1：属性默认为1，如果空间不足，元素缩小。
    • flex-basis: 0%：属性定义在分配多余空间之前，元素占据的主轴空间。浏览器就是根据这个属性来计算是否有多余空间的。默认值为auto，即项目本身大小。设置为0%之后。因为有flex-grow和flex-shrink的设置会自动放大或者缩小。在做两栏布局时，如果右边的自适应元素flex-basis设为auto的话，其本身大小将会是0

    # line-height如何继承？

    • 父元素的line-height写了具体数值，比如30px，则子元素line-height继承该值
    • 父元素的line-height写了比例，比如1.5或2，则子元素line-height也是继承该比例
    • 父元素的line-height写了百分比，比如200%，则子元素line-height继承的是父元素font-size * 200%计算出来的值。
