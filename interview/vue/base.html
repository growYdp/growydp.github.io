    # MVC和MVVM的区别

    MVC

    Model-View-Control 通俗来说就是Control负责将Model获取的数据展示在View上。

    • Model：应用程序中处理应用程序与数据逻辑的部分，模型对象通常负责从数据库中读取或存入数据。
    • View：即负责应用程序中的视图层，负责将Model层返回的数据展示在视图上
    • Control：处理用户交互，通常控制器负责从视图中读取数据，控制用户输入，并向Model层发送用户传入数据。

    MVVM

    Model-View-ModelView

    • ModelView：数据的双向绑定，实现的方式是：DOM事件监听。
      1. 将【模型】转化成【视图】，将后端传递的数据转化成页面
      2. 将【视图】转化成【模型】，将页面转化成后端的数据

    区别

    实现了View和Model的自动同步，也就是当Model的属性改变时，不用再自己手动操作DOM元素，来改变View的显示，而是改变属性后该属性对应View层显示会自动改变（对应Vue数据驱动的思想）

    MVVM比MVC更精简，简化了页面与界面的依赖，还解决了数据频繁更新的问题，不需要手动操作DOM元素。因为在MVVM中，View不知道Model的存在，Model和ViewModel也观察不到View，低耦合模式提高代码可重用性。

    Vue没有完全遵循MVVM的思想：严格的MVVM要求View不能和Model直接通信，而Vue提供$refs这个属性，让Model可以直接操作View，违反了这一规定，所以说Vue没有完全遵循MVVM。

    # Vue的内部指令有哪些？分别说一说？

    指令名称 描述
    v-if/v-else/v-else-if 三元运算符
    v-show 通过控制display控制元素显隐
    v-html innerHTML，注意防止xss攻击
    v-text textContent
    v-model 可以用在普通标签和组件上
    v-for 循环指令
    v-slot 插槽
    v-pre 跳过这个元素以及子元素的编译过程，加快整个项目的编译进度
    v-cloak 解决页面闪动
    v-once 定义它的元素只加载一次，包括元素或组件的所有子节点。首次渲染后，不再随数据的变化重新渲染，将被视为静态内容
    v-bind 绑定属性
    v-on 绑定事件

    # Vue的生命周期方法有哪些？一般在哪一步发送请求？

    create阶段

    • beforeCreate：在new Vue()之后，当前阶段data/methods/computed/watch上的数据和方法都不能访问
    • created：在实例创建后完成。当前阶段已经完成了数据观测、属性和方法的运算、watch/event事件回调。当前阶段不能与DOM进行交互，如果想在此阶段交互需要使用vm.$nextTick来访问DOM

    mount阶段

    • beforeMount：在挂载开始之前被调用，当前阶段虚拟DOM已经创建完成，即将开始渲染。在此阶段修改DOM不会触发updated
    • mounted：挂载完成之后被调用，DOM挂载完毕，完成双向绑定，可以访问到DOM节点，可以使用$refs对DOM进行操作

    update阶段

    • beforeUpdate：发生在更新之前，也就是响应式数据发生更新，虚拟DOM重新渲染和打补丁（patch）之前被触发。在此阶段可以进行更改数据，不会发生重渲染。
    • updated：在更新完成之后，当前阶段组件DOM已经完成更新。应该避免在此阶段修改数据，因为会导致无限循环的更新。

    destroy阶段

    • beforeDestroy：发生在实例销毁之前，此阶段实例完全可使用。一般在此阶段进行善后收尾工作，比如清除定时器
    • destroyed：在实例销毁之后调用，Vue实例指示的所有东西都会解绑定，所有事件侦听器会被移除，所有的子实例会被销毁。

    keep-alive相关

    • activated：组件被激活时被调用
    • deactivated：组件不是活跃的状态时被调用

    一般在created/beforeMount/mounted这三个生命周期阶段时发送请求，因为在这些阶段中data已经创建，可以把从服务端获取的数据赋值到data中。

    最好在created中发起异步请求

    • 能更快获取到服务端数据，减少页面loading时间
    • ssr不支持beforeMount/mounted生命周期，放在created中具有一致性

    # 为什么data是一个函数？？？

    因为函数每次返回一个新对象，每复用一次组件都会返回一个新的data。类似于每个组件实例创建一个私有的数据空间，让各个组件实例维护各自的数据。如果单纯的写成对象的形式，使得所有组件实例都共用一份data，会导致某个组件中修改属性也会影响其他组件。

    # Vue组件通讯有哪几种方式？

    1. props和$emit 父子组件传值
    2. $refs 获取组件实例
    3. $parent $children $root 获取当前组件的父组件和子组件还有根组件
    4. provide inject 隔代组件通信
    5. $attrs $listeners 隔代组件通信 配合inheritAttrs使用
    6. mitt 隔代组件通信
    7. eventBus 隔代组件通信（Vue3不支持，可以使用mitt）
    8. vuex 父子/隔代/兄弟组件通信
    9. slot 父子组件通信

    # v-if和v-show的区别

    • 手段
      • v-if本质上是三元运算符，根据条件动态向DOM树内插入或移除节点
      • v-show通过display控制节点的显隐
    • 编译过程：v-if切换有一个局部编译/卸载的过程
    • 编译条件：
      • v-if是惰性的，如果初始条件为假，则什么也不做；只有第一次为真才会开始局部编译
      • v-show在任何条件下，无论首次条件是否为真，都被编译，然后被缓存，DOM元素保留
    • 性能消耗：
      • v-if有更高的切换消耗
      • v-show有更高的初始渲染消耗
    • 使用场景：
      • v-if适用于不太频繁切换的场景
      • v-show适用于频繁切换的场景

    # 怎样理解Vue的单向数据流？

    在Vue中，数据总是由父组件流入子组件。如果子组件也能随意修改父组件的数据，代码将变得难以理解，原理上更是一团糟。

    在子组件中使用v-model绑定父组件传入的数据在开发环境下是会报错的，

    Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders. Instead, use a data or computed property based on the prop's value. Prop being mutated: "property"

    # computed和watch的区别和运用的场景

    computed

    • 支持缓存，只有依赖的数据发生变化，计算属性的值才会重新计算
    • 不支持异步，当computed中有异步操作时，无法监听数据的变化
    • 如果computed属性的属性值是函数，那么默认使用get方法，函数的返回值就是属性的属性值；在computed中，属性有一个get方法和一个set方法，当数据发生变化时，会调用set方法。

    watch

    • 不支持缓存，当一个属性发生变化时，它就会触发相应的操作
    • 支持异步监听
    • 监听的函数接收两个参数，第一个参数是最新的值，第二个参数是旧值
    • 监听数据必须是data中声明的或者父组件传递过来的props中的数据，当发生变化时，会触发其他操作
    • 函数有两个参数：
      • immediate 组件加载立即触发回调函数
      • deep 深度监听，发现数据内部的变化，在复杂数据类型中使用

    当想要执行异步或者昂贵的操作以响应不断的变化时，就需要使用watch

    # 总结

    • computed计算属性：依赖其它属性值，并且computed的值有缓存，只有它依赖的属性值发生改变，下一次获取computed的值时才会重新计算computed的值。
    • watch侦听器：更多的是观察的作用，没有缓存性，类似于某些数据的监听回调， 每当监听的数据变化时都会执行回调进行后续操作。

    # 使用场景

    • 当需要进行数值计算，并且依赖于其他数据时，应该使用computed，因为可以利用computed的缓存特性，避免每次获取值的时候都重新计算。
    • 当需要在数据变化时执行异步或开销较大的操作时，应该使用watch，使用watch选项运行执行异步操作（访问一个API），限制执行该操作的频率，并在得到最终结果前，设置中间状态。这些都是计算属性无法做到的。

    # v-if和v-for为什么不建议一起使用？

    • Vue2中，v-for和v-if一起使用，v-for的优先级高于v-if
    • Vue3中，v-if和v-for一起使用，v-if的优先级高于v-for

    # computed和methods的区别

    可以将同一函数定义为一个method或者一个计算属性，对于最终的结果，两种方式是相同的。

    不同点：

    • computed计算属性是基于它们的依赖进行缓存的，只有在它的相关依赖发生改变时才会重新求值
    • method调用总会执行该函数

    # slot是什么？有什么作用？原理是什么？

    slot是插槽，是Vue的内容分发机制，组件内部的模板引擎使用slot元素作为承载分发内容的出口。插槽slot是子组件的一个模板标签元素，这一个标签元素是否显示，以及怎么显示是由父组件决定的。slot分为三类：默认插槽、具名插槽和作用域插槽。

    # 默认插槽

    又名匿名插槽，当slot的name值没有指定时，默认显示一个插槽，一个组件内只有一个匿名插槽，name值是default

    // children.vue
<template>
  <main>
    <slot />
  </main>
</template>

    上面我们使用了默认插槽

    // parent.vue
<template>
  <div>
    <children>
      主体内容
    </children>
  </div>
</template>

    上面是默认写法，还有另一种，即显示default

    // parent.vue
<template>
  <div>
    <children>
      <template v-slot:default>
        主体内容
      </template>
    </children>
  </div>
</template>

    # 具名插槽

    有时候需要多个插槽，就需要一个特殊的attribute，即name。一个组件可以出现多个具名插槽。

    // children.vue
<template>
  <div>
    <header>
      <slot name="header" />
    </header>
    <main>
      <slot />
    </main>
    <footer>
      <slot name="footer">
    </footer>
  </div>
</template>

    // parent.vue
<template>
  <div>
    <children>
      <template #header>
        顶部内容
      </template>
      <template>
        主体内容
      </template>
      <template #footer>
        底部内容
      </template>
    </children>
  </div>
</template>

    # 作用域插槽

    在子组件渲染作用域插槽时，可以将子组件内部的数据传递给父组件，让父组件根据子组件的传递过来的数据决定如何渲染该插槽。

    // children.vue
<template>
  <div>
    <slot :user="user">
  <div>
</template>

<script>
export default {
  data() {
    return {
      user: {
        name: 'growydp'
      }
    }
  }
}
</script>

    // parent.vue
<template>
  <div>
    <children>
      <template #default="{ user }">
        {{ user.name }}
      </template>
    </children>
  </div>
</template>

    # 实现原理

    当子组件vm实例化时，获取到父组件传入的slot标签里的内容，存放在vm.$slot中，默认插槽为vm.$slot.default，具名插槽为vm.$slot.xxx。当组件执行渲染函数时，遇到slot标签，使用$slot中的内容进行替换，此时可以为插槽传递数据，如果存在数据，则可称该插槽为作用域插槽。

    # 过滤器的作用

    过滤器是用来过滤数据的，在vue中使用filters来过滤数据，filters不会修改数据，而是过滤数据，改变用户看到的输出。

    过滤器是一个函数，会把表达式中的值始终当做函数的第一个参数，过滤器用在插值表达式{{}} 和 v-bind 表达式中，然后放在操作符|后面进行显示。

    <template>
  <div>{{ price | filterPrice }}</div>
</template>

<script>
export default {
  data() {
    return {
      price: 7
    }
  },
  filters: {
    filterPrice(price) {
      return price ? (`￥${price}`) : '--'
    }
  }
}
</script>
